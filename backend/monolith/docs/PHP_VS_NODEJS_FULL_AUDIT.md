### SQL: Хранение ссылок — КРИТИЧЕСКАЯ РАЗНИЦА

**PHP** обновляет колонку `t` для ссылок:
```sql
UPDATE $z SET t=$v WHERE id=$req_id  -- ссылочный объект в t
```

**Node.js** обновляет колонку `val`:
```sql
UPDATE db SET val = ? WHERE id = ?  -- ссылочный объект в val (НЕПРАВИЛЬНО)
```

> ❌ **КРИТИЧНО:** Ссылки хранятся в неправильной колонке. PHP хранит ID ссылочного объекта в `t`, Node.js — в `val`. Это ломает все ссылочные данные.
